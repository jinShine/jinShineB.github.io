클로저(Closure)

스위프트에서 함수형 프로그래밍을 접할 때 꼭 잘 알아야할 부분이 클로저 입니다. C언어나 Objective-C의 블록(block) 또는 다른 프로그래밍언어의 람다(Lambda)와 유사합니다.

사실 함수는 클로저의 한 형태다!

일반 메서드들은 이름을 가지는 클로저의 한 형태이고, 흔히 클로저는 이름을 가지지 않는 익명 함수를 말합니다.

클로저는 1급 객체 입니다.

1급 객체 (First-class citizen)

  • 변수나 데이터에 할당할 수 있어야 한다.
  • 객체의 인자로 넘길 수 있어야 한다.
  • 객체의 리턴값으로 리턴할 수 있어야 한다.

클로저 표현식

먼저 함수의 표현식을 살펴 보겠습니다.

함수는 func,함수 이름,parameters,ReturnType,(),{},-> 이런 조합들로 이루어져 있습니다.

클로저는 일반 함수와 달리 func 키워드 대신 in키워드를 사용하고, 함수 이름이 없는 차이가 있습니다.

클로저 표현식으로 한번 살펴 보겠습니다.

위의 표현식처럼 클로저는 { }로 시작하며, parameters,ReturnType은 함수와 동일하며 in키워드는 실행할 코드와 구분하는 역할을 합니다.
함수에선 { }에 실행할 코드를 넣어주어 구분이 필요 없었지만 클로저는 모든 요소가 { }사이에 존재하기 때문에 in과 같은 키워드가 필요 합니다.

파라미터 생략이 가능합니다.

리턴 생략이 가능합니다.
리턴을 생각할땐 화살표도 모두 없애줍니다.

파라미터와 리턴이 모두 없다면 모두 생략하고 { }와 코드만 사용할 수 있습니다.

Syntax Optimizations

위에 코드들을 실행과 동시에 c의 파라미터를 넣은 형태
인라인 클로저 형태

c파라미터를 통해 파라미터 유추 가능.
파라미터 이름을 제외한 나머지 요소를 삭제 가능

리턴값 생략 가능

파라미터의 갯수를 유추할 수 있다면 Shorthand Argument Name($와 숫자로 표현된 형태)으로 대체 가능
Shorthand Argument Name를 사용할 경우 파라미터 이름과 in을 모두 생략 가능 합니다.

이로써 클로저 문법을 최대로 간소화 되서 표현을 사용할 수 있습니다.
위와 같은 문법은 인라인클로저형태이며 후행클로저로 표현해서 사용할 수 있습니다.

후행 클로저(Trailing Closure)

클로저가 조금 길어지거나 가독성이 조금 떨어진다 싶으면 후행 클로저사용하면 좋습니다.

단, 후행 클로저는 맨 마지막 전달인자로 전달되는 클로저에만 해당되므로 전달인자로 클로저가 여러 개 전달될 때에는 맨 마지막 클로저만 후행 클로저로 사용할 수 있습니다.

code라는 argument name을 선언해보았습니다.

argument로 전달할 코드가 ()뒤로 이동합니다.

괄호에 포함될 argument이름은 생략된다.
만약 다른 파라미터가 있다면 ()를 생략 불가능합니다.

후행클로저는 맨 마지막 전달인자로 전달되는 클로저에만 해당된다고 했습니다.
파라미터가 여러개일때를 확인해 보겠습니다.

값 획득 (Capturing Values)

클로저는 자신이 정의된 위치에 주변의 상수나 변수를 획득(Capturing)할 수 있습니다.
Capturing을 통해 클로저는 주변에 정의된 상수나 변수가 더는 존재하지 않더라도 그 상수나 변수의 값을 자신 내부에서 참조하거나 수정할 수 있습니다.

클로저는 비동기 작업에 많이 사용되는데 비동기 콜백의 형태를 작성하는 경우, 현재 상태를 미리 획득해 두지 않으면, 실질적으로 클로저의 기능을 수행하는 순간에는 주변의 상수나 변수가 이미 메모리에 존재하지 않는 경우가 발생할 수가 있기 때문입니다.

값을 획득 하는 방식은 크게 2가지가 있습니다.

  1. 값이 획득될때 클로저 내부에 값이 복사가 되어 전달 되는 방식
  2. 주소값을 참조 하는 방식

1.Value Type Capture

2.Reference Type Capture

탈출 클로저

함수의 파라미터로 전달된 클로저가 함수 종료 후에 호출될 때 클로저가 함수를 탈출(Escape)한다고 표현합니다.
클로저가 매개변수로 갖는 함수를 선언할 때 매개변수 이름의 콜론(:) 뒤에 @escaping키워드를 사용하여 클로저가 탈출하는 것을 허용한다고 명시해줄 수 있습니다.

@escaping을 따로 명시하지 않는다면 매개변수로 사용되는 클로저는 기본적으로 비탈출(Nonescape)클로저 입니다.

탈출클로저는 함수가 종료되고 난 후 작업이 끝나고 호출될 필요가 있는 클로저를 사용해야 할 때 필요.
비탈출클로저는 함수의 동작이 끝난 후 전달된 클로저가 필요 없을때 사용합니다.

결과적으로 1 -> closure -> 2가 실행되는걸 볼 수 있습니다.
위의 코드는 비탈출코드입니다.
위는 코드는 아니지만 탈출이라는것은 클로저의 실행주기가 함수의 실행주기는 벗어난다는 것입니다.

자동클로저(AUtoClosure)

함수의 전달인자로 전달되는 표현을 자동으로 변환해주는 클로저를 자동클로저(Auto Closure)라고 합니다.
자동 클로저는 전달인자를 갖지 않습니다. 호출되었을때 자신이 감싸고 있는 코드의 결과값을 반환하고
함수로 전달되는 클로저를 어려운 클로저 문법을 사용하지 않고도 클로저로 사용할 수 있도록 문법적 편의를 제공합니다.
@autoclosure키워드를 사용합니다.
