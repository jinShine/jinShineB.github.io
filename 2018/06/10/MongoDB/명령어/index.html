기본 명령어

들어가기에 앞서, mongo Shell에서 쓰이는 문법은 기본적으로 Javascript로 되어 있습니다.

서버실행

mongod

서버 접속

mongo

서버 접속 까지 완료해줍니다.

DB 생성

use DATABASE_NAME명령어를 통해서 Database를 생성할 수 있습니다.

> use testDB

switched to db testDB

testDB라는 이름의 데이터베이스를 생성.

현재 사용중인 DB 보기

db

> db

testDB

DB 리스트들 보기

show dbs

> show dbs

local 0.000GB

방금 만든 testDB는 어디갔지?
최소 한개의 Document를 추가해야 리스트에서 볼 수 있습니다.

일단 데이터를 한번 넣어볼까요?
지금 명령어는 나중에 다 알아 볼겁니다!

use testDB
db.user.insertOne({username:“seungjin”,password:“111111”})
db.course.insertOne({name:“mongoDB”,grade:1})

다 되셨나요?

데이터를 넣을때 이런식으로 리턴값이 나오면 잘 들어간겁니다!

이제 show collections라는 명령어를 한번 입력해 봅시다.

user와 course가 보이시나요?
미리 알수 있듯이 안에 testDB안에 Collection들 입니다.

db는 현재 DB를 가리키고, user나 course는 collection을 나타냅니다.

db.collection.command

이런 형식으로 command부분에 쿼리를 작성합니다.

DB안의 Collection 리스트 보기

show collections

Collection안의 Document 보기

db.user.find()

좀 더 예쁘게 보고 싶다!!

db.user.find().pretty()


Create 명령어

Document를 MongoDB에 넣는 방법에 대해 알아 보겠습니다.

하나의 데이터를 입력 하고싶을 때

db.collection.inserOne({document1})

>db.user.insertOne({username:“seungjin”,password:“111111”})

여러개의 데이터를 입력 하고싶을 때

db.collection.inserMany([{document1},{document2},…])

>db.collection.insertMany(
{username:“seungjin”,password:“11”},
{username:“Hi”,password:“123”},
{username:“hello”,password:“12345”}
)


Read 명령어

데이터베이스로부터 데이터를 불러오는 기본직인 방법에 대해 알아 보겠습니다.

Collection안의 Document 보기

db.user.find()

위와 같은 find()명령어로 Document리스트를 읽을 수 있다고 했습니다.

find()안에 BJON형태의 Query가 들어갑니다. 즉, 조건에 맞는 값만 불러올 수 있습니다.

{field:value, field:value,…}

> db.user.find({username:“seungjin”})

username이 seungjin인 Document를 전부 가져옵니다.

원하는 Document 보기

또 옵션을 줄 수 있습니다.

{ “_id” : ObjectId(“ObjectId”), “username” : “seungjin”, “password” : “111111” }

이런 Document가 있습니다.

> db.user.find({username:“seungjin”})

username이 seungjin인 Document를 전부 가져온다고 했을때 username으로 값을 찾지만 password데이터만 보여 주고 싶을때의 옵션을 줄 수 있습니다.

{field:boolan, field:boolean,…}
boolean이 true이면 포함한 field를 표현하고, false이면 제외한 결과를 출력합니다.

>db.user.find({username:“seungjin”},{password:true})

{ “_id” : ObjectId(“ObjectId”), “password” : “111111” }
